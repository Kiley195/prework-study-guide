HTML

  • The head element contains information abour the webpage.
  • The body element represents the visible content shown to the user.
  • Child elements are elements inside of other elements.
  • Text, image, header, and list elements are used to add content to a page.
  • Meta-related, content division, and semantic elements divide and describe a page.
  • Metadata helps the browser render the page correctly, it is not seen by users.
  • Headers are h1 - h6, the numbers represent the level of heading. These should be used in order and h1 should only be used once per page.
  • Semantic HTML elements: the name of the element indicates role of content as well as providing structure to the content, these should be used where ever possible.
  • When adding an image the 'src' attribute defines it's location or source; the 'alt' attribute contains a text string the describes the image. The 'alt' text is important for users using assistive technologies and to help improve search engine rankings.
  • 'Section' elements define a sectio in a document that contains thematic content.

Git

  • cd "name of directory": navigate to that directory
  • git status:checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone [url]: retrieve an entire repository from a hosted location via URL
  • git add -A: add modifications made in the current working branch to the staging area
  • git commit -m "[descriptive message]": commit staged content
  • git pull origin "branch you want to pull from": pulls changes from base branch to local environment - ensures you have the latest version of the project on your local pc
  • git push origin "branch you want to push to" : push changes made locally to remote GitHub branch
